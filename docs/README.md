# Dokumentenordnung

Diese Datei ordnet die Hintergrunddokumente im Projekt.

## Bereiche

- analysis/: Systemanalysen, Fragebögen und Review-Dokumente
- reference/: Nachschlagewerke, Begriffe und Konventionen
- testplan/: Testplan-Zusammenfassungen und Ergebnis-Präsentationen
- ../reports/: operative Runbooks, KPI-Reports und Monitoring-Artefakte

## Schnellzugriffe

- analysis/SYSTEM_ANALYSE_AUSWERTUNG.md
- analysis/SYSTEM_ANALYSE_FRAGEBOGEN.md
- reference/NOMENKLATUR.md
- testplan/TESTPLAN_SUMMARY.md
- testplan/TESTPLAN_RESULTS_PRESENTATION.md

## Ablageregel

Wenn ein Dokument für den laufenden Betrieb nötig ist, bleibt es im Root oder unter reports/.
Wenn es vor allem Analyse, Review oder Entscheidungsdokumentation ist, gehört es unter docs/.
